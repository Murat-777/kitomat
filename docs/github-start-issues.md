# Start-Issues für GitHub / Codex

## Issue 1: AGENTS.md im Repository ergänzen

### Ziel
Codex soll klare Projektregeln erhalten, bevor technische Änderungen durchgeführt werden.

### Aufgabe
Lege die Datei `AGENTS.md` im Repository-Root an.

### Akzeptanzkriterien
- [ ] AGENTS.md liegt im Root.
- [ ] Regeln zu minimal-invasiven Änderungen sind enthalten.
- [ ] Verbot von Rewrites und Funktionsentfernung ist enthalten.
- [ ] PR-Format und Definition of Done sind enthalten.

---

## Issue 2: README um KI-/Lean-Projektstruktur ergänzen

### Ziel
Das Repository soll verständlich erklären, welche Rollen ChatGPT Project, Codex und Quellenprüfung haben.

### Aufgabe
README um Abschnitt „Agenten-Workflow“ ergänzen.

### Akzeptanzkriterien
- [ ] Rollenverbund wird erklärt.
- [ ] Codex ist als technischer Agent beschrieben.
- [ ] ChatGPT Project ist als Konzept- und Review-Ort beschrieben.
- [ ] Quellenprüfung ist separat beschrieben.
- [ ] Keine bestehenden README-Inhalte werden entfernt.

---

## Issue 3: Review-Canvas als Markdown-Vorlage ergänzen

### Ziel
Fachliches Review soll standardisiert werden.

### Aufgabe
Lege `docs/review-canvas-template.md` an.

### Akzeptanzkriterien
- [ ] Canvas enthält Modul, Lean-Bezug, KI-Rolle, Trust-Level, Review-Fragen und Risiken.
- [ ] Vorlage ist ohne weitere Erklärung nutzbar.
- [ ] Keine bestehenden Dateien werden verändert, außer Verlinkung in README falls sinnvoll.

---

## Issue 4: Quellenstatus im Dashboard sichtbarer machen

### Ziel
Nutzer sollen erkennen, ob Aussagen dokumentiert, teilweise dokumentiert oder noch offen sind.

### Aufgabe
Bestehende Trust-/Quellenstatus-Logik analysieren und minimal verbessern.

### Akzeptanzkriterien
- [ ] Bestehende Statuswerte bleiben erhalten.
- [ ] Sichtbarkeit oder Erklärung wird verbessert.
- [ ] Keine fachliche Aussage wird ohne Quelle aufgewertet.
- [ ] Änderung ist klein und im PR beschrieben.

---

## Issue 5: Szenario-Review für ein Lean-Modul ergänzen

### Ziel
Ein konkretes Lean-Modul soll mit Review-Fragen und Risiken erweitert werden.

### Aufgabe
Wähle ein bestehendes Modul und ergänze Review-Fragen, Grenzen und Quellenstatus.

### Akzeptanzkriterien
- [ ] Nur ein Modul wird angepasst.
- [ ] Review-Fragen sind praxisnah.
- [ ] Risiken sind sichtbar.
- [ ] Bestehende Modulstruktur bleibt erhalten.
